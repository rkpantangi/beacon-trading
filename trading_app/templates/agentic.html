Agentic Trading

Connect your AI coding assistants and IDEs directly to your Beacon Trading paper trading account.

Model Context Protocol (MCP)

Beacon Trading supports the Model Context Protocol (MCP), an open standard that allows AI models and coding assistants (like Google Antigravity, Claude Code, and Cursor) to securely access local or remote tools.

By connecting the Beacon Trading MCP server, your AI agent can:

  • 📊 Query Market Data: Get real-time stock quotes.
  • 💼 Manage Portfolio: View your account balances and current holdings.
  • 📈 Place Trades: Execute market buy and sell orders.
  • 📝 Order Book: List, review, and cancel active limit or market orders.

1. Cloud Integration (Recommended)

Use this setup if you want to connect your local terminal or desktop agent to your cloud-hosted Fly.io trading engine.

Step A

Ensure your cloud MCP server is deployed and running. Run the following command in your terminal to add the server to your agent's config:

Terminal Command 
claude mcp add --scope user --transport http beacon-trading https://agent-beacon-trading-paper.fly.dev/mcp
Step B

Restart or reload your agent session to activate the tools. In the Antigravity TUI, type /mcp to refresh your connections.

2. Local Development Integration

If you are running the trading stack locally on your computer, you can configure your agent to spin up the local MCP server process directly via stdio.

Terminal Command (Local Stdio) 
claude mcp add --scope project --transport stdio beacon-trading-local -- uv run --directory ./mcp_server beacon-trading-mcp-server

Note: Replace ./mcp_server with the absolute path to your local mcp_server folder if running outside the project root.
